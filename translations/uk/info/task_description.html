Роботи вирішили покопатися в собі і може бути навіть поліпшити що-небудь. У цьому завданні Вам потрібно написати свою реалізацію вбудованих функцій (версії для py3) min і max . Деякі вбудовані функції заблоковані тут: import, eval, exec, globals. Не забувайте, що в цьому завданні вам потрібно реалізувати дві функції, а не одну, як звичайно.

max(iterable, *[, key]) або min(iterable, *[, key])
max(arg1, arg2, *args[, key]) або min(arg1, arg2, *args[, key])

Повертає найбільший (найменший) елемент в ітерованому (iterable) параметрі або найбільше (найменше) з двох і більше аргументів.

Якщо заданий тільки один позиційний аргумент, то він повинен бути ітерованим. У цьому випадку функція повертає найбільший (найменший) елемент з даного ітерованого. Якщо задані два або більше позиційних аргументів, то повертати буде найбільший (найменший) з даних аргументів.

Необов'язковий ключовий аргумент key визначає функцію одного аргументу, яка використовується для вилучення ключа для порівняння з кожного елемента масиву (наприклад, key = str.lower).

Якщо масив містить декілька максимальних (мінімальних) значень, то функція повертає перший по порядку в масиві.

-- Документація Python (Вбудовані функції на англійській мові)

Вхідні дані: Один позиційний аргумент, як ітероване або два або більше позиційних аргументів. Необов'язковий ключовий аргумент, як функція.

Вихідні дані: Найбільший елемент для функції "max" і найменший для функції "min".

Приклади: 

max(3, 2) == 3
min(3, 2) == 2
max([1, 2, 0, 3, 4]) == 4
min("hello") == "e"
max(2.2, 5.6, 5.9, key=int) == 5.6
min([[1,2], [3, 4], [9, 0]], key=lambda x: x[1]) == [9, 0]

Як це використовується: Це завдання може допомогти вам краще розуміти, як працюють вбудовані функції на більш глибокому рівні.

Передумови: Всі тести коректні, згідно опису функції і не повинні породжувати винятку.
